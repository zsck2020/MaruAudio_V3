| 焦点环 | `--color-focus-ring` 全局 2px ring | a11y |
| 字体 | HarmonyOS Sans SC 离线内嵌 | 中文友好 |

### 优点

1. **覆盖极广**：开发者几乎不需要写 magic number，所有间距/颜色/阴影/动画都有 token
2. **命名一致**：xs/sm/md/lg/xl/2xl 命名连贯
3. **暗色优化**：5 级背景层级 + 阶差强化，**不依赖边框线分割视觉块**——这是真正掌握暗色设计的体现
4. **可访问性**：focus-visible 全局 ring + prefers-reduced-motion 完整支持
5. **字体优化**：font-feature-settings 启用连字 + 比例数字、SVG 几何精度渲染、抗锯齿

### 缺点

- 第 130-131 行：`--spacing-xxl: 32px` 与 `--spacing-xl: 32px` **同值不同名**（兼容旧名）
- 第 137 行：`--control-height-md: 40px` 与 `--control-height: 40px` **同值不同名**
- 第 76 行 `--transition-duration: 0.22s` 与第 159-161 行 `--motion-duration-fast/mid/slow` **两套并存**
- 注释写"5 级背景层级"实际定义了 6 个（deepest/base/container/elevated/spotlight/popover）

**修复建议**：清理冗余别名，或在注释里明确"兼容遗留代码"。

### 整体观察

- **Props 类型严格**：每个组件都有 TS Props 接口

## 四、UX 流程问题清单

### 【高】4 项

1. **首次启动引导缺失**：后端 TTS server 未运行时，配音页三引擎全红 + 参考音频检测失败 + 字幕调用直接 connect refused——**没有"启动后端"的引导卡 + 一键启动按钮**。新用户首次打开会一脸懵
2. **死按钮泛滥**（见技术审查报告 #21/#23/#11/#12）：拼音/数字/录音/字幕跳转/PresetLibrary 全线/云端任务 3 控件/默认值按钮——用户视角"看着能用但点了没反应"达 **15+ 处**
3. **Mock 数据未标记**：用户中心、关于、音库列表、文件列表、首页"12,840 字符"等全是写死的，未来对接真实数据时易混淆
4. **DEV banner 遮挡 TitleBar**：开发期可以接受，但调试体验差

### 【中】5 项

5. **窄屏适配未做**：1024px 以下挤压严重（首页 4 卡变成单字竖排 · 设置页文字断行）
6. **引擎状态点反馈不够细**：三引擎红点只显示"不可用"，应在 tooltip 显示具体原因（"TTS 服务未启动" vs "未找到模型文件" vs "显存不足"）
7. **缺 loading 骨架屏**：首页打开瞬间 banner / 最近项目 / 今日字符 都没有 skeleton placeholder，会有明显闪烁
8. **进度反馈不连续**：推理时只有进度条，缺"剩余时间预估"或"已生成 x 秒音频"
9. **toast 一闪而过**：错误信息过短不易看清，应允许鼠标悬停延迟消失

### 【低】4 项

10. **配音页拖放区无视觉反馈**：dragover 期间已加了 .dragging 类，但 drop 后**没有"识别中"反馈动画**
11. **快捷键 Ctrl+Shift+E 切换引擎**：toast 提示了，但**没有可视化菜单浮现**（不知道循环到第几个）
12. **响应式断点保守**：768px 才切 mobile，但 1024px 的窗口就已经挤压了
13. **设置页右侧"选择左侧章节进行配置"占位文字过大**：占满半屏，应改用 illustration + 提示
