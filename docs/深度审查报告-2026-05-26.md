### S1 · 生产 CSP 与前端 fetch 矛盾（连锁挂机）

**文件**：`frontend/src-tauri/tauri.conf.json` + `frontend/src/lib/api/tts.ts`

```24:27:frontend/src-tauri/tauri.conf.json

      "csp": "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' https://auth.wzagent.cn data:; connect-src 'self' https://auth.wzagent.cn wss://auth.wzagent.cn; font-src 'self' data:; media-src 'self' blob:; worker-src 'self' blob:"
```

`connect-src` 只允许 `'self'`（即 tauri://localhost）+ auth.wzagent.cn。但前端有 **7 处 fetch 直连** 9880：

|---|---|---|---|

| `vocalSeparateInfo()` | 184 | 人声分离能力检测 | 配音 + 音库 |
| `vocalSeparate()` | 289 | 人声分离 SSE | 配音 |
| `transcribe()` 回落 | 342 | 字幕 SSE（dev 走桥接、prod 桥接） | 字幕 |
| `listPresets()` | 442 | 预置样音 | 音库 |
| `listLlmModels()` | 455 | LLM 模型列表 | 设置 |
| `splitLines()` | 493 | LLM 台词拆分 | 多角色 |

仅 `transcribe()` 通过 `isTauri()` 自动走 Rust 桥（`subtitle_transcribe_stream` 命令），其余 6 个在 prod 打包都会被 CSP 拦截 → console 报 `Refused to connect`。

```39:45:frontend/src-tauri/src/commands/subtitle.rs
    Ok(content)
```

### S4 · Python `/llm-models` / `/split-lines` SSRF

**文件**：`backend/maruaudio_tts/server.py:386-413, 427-497`

`api_base_url` 由前端任意传入，server.py 用 httpx 直接请求。Python server 跑在 127.0.0.1，**虽然 CORS 限制外部，但桌面端 webview 内任何 JS 都能调用**。

#### PoC

```bash

curl -X POST http://127.0.0.1:9880/llm-models \
  -H "Content-Type: application/json" \
  -d '{"api_base_url": "http://169.254.169.254/latest/meta-data", "api_key": "any"}'
# 在 AWS EC2 上会回显元数据；在内网环境会请求 192.168.x.x 设备
```

#### 修复

校验 `api_base_url`：
- 必须是 `https://` 开头
- 域名必须在白名单（deepseek.com / openai.com / dashscope.aliyuncs.com 等）
- 拒绝 IP 地址 + 拒绝内网域名（169.254 / 10.x / 172.16-31 / 192.168.x）

```python
from urllib.parse import urlparse
import ipaddress

ALLOWED_HOSTS = {"api.deepseek.com", "api.openai.com", "dashscope.aliyuncs.com",
                 "generativelanguage.googleapis.com"}

def _validate_llm_url(url: str) -> None:
    parsed = urlparse(url)
    if parsed.scheme != "https":
        raise HTTPException(400, "仅允许 HTTPS")
    host = parsed.hostname or ""
    try:
        ip = ipaddress.ip_address(host)
        if ip.is_private or ip.is_loopback or ip.is_link_local:
            raise HTTPException(400, "禁止内网/回环地址")
    except ValueError:
        pass  # 是域名
    if host not in ALLOWED_HOSTS:
        raise HTTPException(400, f"不在白名单：{host}")
```

### S5 · `_cloud_tmp_files` 全局竞争 → 跨请求文件误删

**文件**：`backend/maruaudio_tts/server.py:26-54`

```26:54:backend/maruaudio_tts/server.py
_cloud_tmp_files: list[str] = []  # 模块级全局
def _decode_base64_audio(req):
    ...
    _cloud_tmp_files.append(tmp.name)
def _cleanup_cloud_tmp():
    while _cloud_tmp_files:
        path = _cloud_tmp_files.pop()
        try: os.unlink(path)
        except OSError: pass
```

**并发场景**：
- 请求 A 解码 → 加入 tmp_A
- 请求 B 解码 → 加入 tmp_B  
- 请求 A 完成 → `_cleanup_cloud_tmp()` 把 tmp_A 和 tmp_B **全删了**
- 请求 B 还在用 tmp_B → ffmpeg / IndexTTS 读取失败

#### PoC

```python
import asyncio, httpx
async def cloud_call(text):
    async with httpx.AsyncClient(timeout=120.0) as c:
        return await c.post("http://127.0.0.1:9880/synthesize/stream",
            json={"engine":"cloud","text":text,"speaker_audio_path":"x",
                  "speaker_audio_base64":"<base64>","emotion_method":"slider", ...})

asyncio.gather(cloud_call("A"), cloud_call("B"))  # 并发两个
# 50% 概率 B 报 "no such file" 因为 A finally 把 tmp_B 也删了
```

#### 修复

用 contextvars 或请求级 list：

```python
from contextvars import ContextVar
_tmp_ctx: ContextVar[list[str]] = ContextVar("cloud_tmp", default=None)

def _decode_base64_audio(req):
    if _tmp_ctx.get() is None:
        _tmp_ctx.set([])
    ...
    _tmp_ctx.get().append(tmp.name)

def _cleanup_cloud_tmp():
    files = _tmp_ctx.get() or []
    for path in files:
        try: os.unlink(path)
        except OSError: pass
    _tmp_ctx.set([])
```

## 三、【高】8 项

### H1 · Tauri capabilities `fs:` 权限过宽

**文件**：`frontend/src-tauri/capabilities/default.json:18-23`

```18:23:frontend/src-tauri/capabilities/default.json
    "fs:default",
    "fs:allow-read-file",
    "fs:allow-write-file",
    "fs:allow-exists",
    "fs:allow-mkdir",
    "fs:allow-appdata-read-recursive",
    "fs:allow-appdata-write-recursive",
```

`fs:allow-read-file` / `fs:allow-write-file` 是 **全局** 文件权限，**没有 scope**。前端 `@tauri-apps/plugin-fs` 调用 `readFile('C:/任意路径')` 都通过。

**修复**：加 `fs:scope`，限定到 output、ref_audio、appdata 等目录。

### H2 · CSP `style-src 'unsafe-inline'`

`style-src 'self' 'unsafe-inline'` 允许内联样式 → 一旦 LLM 返回内容含恶意 style 属性注入 → XSS。

**修复**：用 CSS 文件 + nonce 替换内联样式。

### H3 · Python 错误信息全暴露

`raise HTTPException(status_code=500, detail=str(e))` 在 server.py 出现 6 次。生产环境会把 Python 异常路径、堆栈信息泄露给前端。

**修复**：包一层 `_sanitize_error()`，仅返回 generic 消息 + 内部 log。

### H4 · Cloud engine 超时设置不一致

| 文件 | 超时 |
|---|---|
| `cloud_engine.check_availability` | 5s |
| `cloud_engine.synthesize_stream` | 600s |
| `cloud_engine.synthesize_sync`（间接） | 600s |
| `engine_manager.check_cloud` thread pool | 6s |

`check_cloud` thread pool 6s 比 inner 5s 略宽（合理），但其他依赖外部资源（GPU 实例可能冷启动 30s+）的 600s 略短。**建议改用配置项**。

### H5 · 重复实现的 settings store

`lib/stores/settings.svelte.ts`（带 llm）和 `lib/overrides/settings-store.override.svelte.ts`（不带 llm）**两个并存**。

- 后者通过 `lib/overrides/settings-core.ts` 提供 `createSettingsState` 重用
- 实际加载哪一个取决于 import 路径，**潜在双源同步 bug**

**修复**：删除一个，统一用法。

### H6 · `dubbing/+page.svelte` `max_mel_tokens: 600` 硬编码

```84:84:frontend/src/routes/dubbing/+page.svelte
        max_mel_tokens: 600,
```

而 `cloud_engine.py` 默认 1500，`inference.py:197` 又有 `mel_tokens = req.max_mel_tokens if req.max_mel_tokens > 600 else 1500` 自适应。**单引擎页配音时硬塞 600 会让 v2.0 自动降级**。

**修复**：改读 `dubbing.maxMelTokens` 或按引擎切默认值。

### H7 · `engine_manager.check_cloud` asyncio 调用栈晦涩

```187:194:backend/maruaudio_tts/engine_manager.py
        try:
            loop = asyncio.get_event_loop()
            if loop.is_running():
                ...
                future = pool.submit(asyncio.run, cloud_engine.check_availability())
                # ↑ 传给 submit 的是 coroutine 对象，asyncio.run 会再 wrap
                # 实际能跑但写法极易触发 RuntimeWarning
```

**修复**：改用 `asyncio.run_coroutine_threadsafe(coro, loop)` 或单独的事件循环。

### H8 · `services/vocal_separator.py` 单例 + GPU 模型多请求竞争

`_singleton: VocalSeparatorService` 全局共享，`_separate_blocking` 在 `loop.run_in_executor(None, ...)` 默认线程池跑。多个并发请求会同时操作 demucs / torch.cuda 状态 → **OOM / 中间结果错乱**。

**修复**：加 `asyncio.Lock` 或自建 `ThreadPoolExecutor(max_workers=1)`。
