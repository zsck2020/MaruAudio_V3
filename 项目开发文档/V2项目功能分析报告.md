## 🎯 核心功能模块

### 1. 配音生成 (Voice Clone)

**位置**: `frontend/pages/voice_clone_page.py` + `backend/core/audio_generator.py`

**功能特性**:
- ✅ **文本转语音 (TTS)**: 基于参考音频进行语音克隆
- ✅ **双引擎支持**:
  - 本地引擎: Qwen3-TTS (0.6B/1.7B模型，根据显存自动选择)
  - 云端引擎: 阿里云百炼 CosyVoice API
- ✅ **大文本处理**: 超过1万字符自动分批处理
- ✅ **文本预处理**: 自动分句、停顿、拼音纠错
- ✅ **参数调节**: 语速、音调、情感控制
- ✅ **参考音频管理**: 支持上传、录音、从样音库选择
- ✅ **参考文本识别**: 自动识别参考音频的文本内容 (ASR)
- ✅ **批量生成**: 支持长文本自动分批生成并合并

**关键类**:
- `VoiceClonePage`: 前端页面
- `AudioGeneratorManager`: 音频生成管理器 (单例)
- `AudioGeneratorWorker`: 后台生成线程
- `LargeTextGeneratorWorker`: 大文本生成线程

**工作流程**:
1. 用户输入文本 + 选择参考音频
2. 前端调用 `AudioGeneratorManager.create_worker()`
3. Worker 在后台线程调用 TTS 引擎生成音频
4. 通过信号槽机制更新 UI 进度
5. 生成完成后加载到音频播放器

### 3. 字幕生成 (Subtitle Generation)

**位置**: `frontend/pages/subtitle_page.py` + `backend/subtitle/`

**功能特性**:
- ✅ **视频/音频转录**: 支持视频和音频文件转文字
- ✅ **ASR引擎**: 必剪 ASR (免费)
- ✅ **字幕翻译**: 支持多语言翻译
- ✅ **字幕格式**: 支持 SRT、VTT、ASS 等格式
- ✅ **字幕编辑**: 时间轴调整、文本编辑
- ✅ **批量处理**: 支持批量文件处理

**关键类**:
- `SubtitlePage`: 前端页面
- `transcribe_audio()`: 音频转录入口
- `transcribe_video()`: 视频转录入口
- `BcutASR`: 必剪 ASR 实现

**工作流程**:
1. 用户上传视频/音频文件
2. 如果是视频，先提取音频 (FFmpeg)
3. 调用 ASR 引擎转录音频
4. 生成字幕文件 (SRT/VTT/ASS)
5. 可选：翻译字幕

## 🔧 技术实现细节

### 1. 单实例应用

**实现**: `main.py` 中使用 `QSharedMemory` 检测是否已有实例运行

```python
SHARED_MEMORY_KEY = "MaruAudio_SingleInstance_Lock_v1"
_shared_memory = QSharedMemory(SHARED_MEMORY_KEY)
```

### 2. 后台线程处理

**音频生成**: 使用 `QThread` 在后台线程执行，避免阻塞 UI

```python
class AudioGeneratorWorker(QThread):
    generation_started = Signal()
    progress_updated = Signal(int, str)
    generation_completed = Signal(str)
    generation_failed = Signal(str)
```

### 3. 信号槽机制

**跨线程通信**: 使用 Qt 信号槽机制实现线程间通信

```python
worker.generation_completed.connect(self._on_generation_completed)
```

### 4. 配置持久化

**用户配置**: 保存在 `%APPDATA%/MaruAudio/user_session.json`

**引擎配置**: 保存在本地配置文件

### 5. 环境检测

**启动时检测**:
- 虚拟环境 (venv)
- 模型文件 (Qwen3-0.6B / Qwen3-1.7B)
- FFmpeg
- Demucs 模型

**GPU 检测**: 使用 `nvidia-smi` 检测显卡和显存

### 6. 性能优化

- **延迟加载**: 引擎在后台线程初始化
- **进度节流**: 避免频繁更新 UI
- **事件处理**: 使用 `QApplication.processEvents()` 保持 UI 响应
- **资源清理**: 及时清理临时文件和模型资源

## 📊 数据流

### 配音生成流程

```
用户输入文本 + 参考音频
    ↓

    ↓
```

## 📝 总结

### 项目特点

1. **完整的桌面应用**: 基于 PySide6 的现代化 UI
2. **双引擎支持**: 本地 + 云端，灵活切换
3. **完善的用户系统**: 注册、登录、会员管理
4. **丰富的功能**: TTS、字幕、文件管理
5. **良好的用户体验**: 进度显示、错误处理、性能优化

### 技术亮点

1. **多线程处理**: 后台线程执行耗时操作，保持 UI 响应
2. **信号槽机制**: 优雅的跨线程通信
3. **引擎热切换**: 运行时切换引擎，无需重启
4. **大文本处理**: 自动分批处理长文本
5. **单例模式**: 统一管理引擎和用户状态

### 迁移到 Tauri + Vue 的考虑

1. **前端**: PySide6 → Vue 3 (需要重新实现所有 UI 组件)
2. **后端**: Python → Rust (Tauri Commands)
3. **AI 引擎**: 可能需要通过 Rust FFI 调用 Python，或完全用 Rust 重写
4. **文件系统**: Tauri 提供安全的文件系统 API
5. **网络请求**: Tauri 提供 HTTP 客户端
6. **系统集成**: Tauri 提供系统托盘、窗口管理等 API
