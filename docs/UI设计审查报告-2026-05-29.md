## 一、设计系统（底座）· ⭐⭐⭐⭐⭐

`app.css` 是一套真正的 design system，而非样式堆砌：

- **5 级背景层级**：deepest → base → container → elevated → spotlight，以阶差表达深度，避免线条分割。
- **10 阶主色**（声波蓝 #3b6eaf）+ 声纹紫 accent + 功能色全套 hover/bg 变体 + 信息色。
- **控件高度体系**：xs 28 / sm 36 / 40 / lg 48 / xl 56；图标尺寸、字阶（xs 12 → 4xl 38）、字重档位齐全。
- **3 级阴影** + **5 种语义化缓动曲线**（base/out/in/in-out/out-back）。
- **全局工程**：`focus-visible` 焦点环、`prefers-reduced-motion`、细滚动条（早期「全隐藏」问题已修）、字体渲染优化（antialiased/liga/tabular-nums）、`::selection`。

## 二、组件库 · ⭐⭐⭐⭐

- `Button`：7 变体 + 3 尺寸 + loading/disabled/block/icon + focus-ring + active 缩放，工程完善。
- **原已存在缺陷（本次已修）**：`component-spec.mdc` 文档写 variant=`default/primary/ghost/link/danger/warning/success`，实际为 `default/primary/text/link/danger/ghost/dashed`——`warning/success` 根本不存在，`text/dashed` 未录入文档，会误导后续开发。
- **原缺口（本次已补）**：缺统一 `Select` / `Segmented` / `LockedAction`，导致各页手写原生 `select` + 分页按钮。

## 三、页面视觉

### 首页 · ⭐⭐⭐⭐
Hero 三屏轮播 + 4 快捷卡 + 仪表盘（最近项目表 + 统计/引擎状态）+ 通知条，层次清晰；引擎状态真实（checkHealth）、额度真实（membership 派生）。
- **原痛点（本次已修）**：最近项目硬编码假数据 + 分页写死（1/2 无 onclick）+「打开」假跳转 → 信任损耗。

### 配音页 · ⭐⭐⭐⭐
文本工具栏 + 大编辑区 + 底部（推理模式/导入/字数/今日剩余）+ 波形播放器，主链路清晰。**配音区配色为硬约束，保持不动。**

### 多角色页 · ⭐⭐⭐⭐⭐（标杆）
角色色编码贯穿（头像/台词卡左色条/标签），台词卡信息层次（角色/情绪/强度/状态 chips + 文本 + 播放/删除），商业差异化最强。
- 可优化：台词卡偏高（~150px），长剧本滚动冗长；播放/删除可行内紧凑化。

### 设置页 · ⭐⭐⭐⭐
左章节导航 + 内容区 row（label + control），信息密度偏高。设置页死循环 bug 此前已修，本次迁移后仍稳。

## 四、跨页设计问题

1. **演示数据混入真实界面**：首页最近项目等，未就绪数据应给明确空态而非假数据。
2. **死控件**：首页分页（已修）、部分页「开发中」toast 按钮；未上线功能宜归拢「实验室/路线图」。
3. **窄视口伸缩**：多栏页右参数栏在窄宽下易裁切/换行，右栏优先级需明确。
4. **长列表**：多角色台词、音库列表数据量大后卡顿（分页已缓解，可上虚拟列表 + 派生索引）。
5. **可发现性**：内部滚动区缺「可滚动」提示（渐隐/阴影）。
6. **a11y 深度**：卡片整体可点的键盘可达性、aria-label、轮播读屏支持需逐一补。
