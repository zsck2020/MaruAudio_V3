# 丸子配音 V3 · 前端深度审查报告

> **审查日期**：2026-05-27  
> **审查范围**：`frontend/src/routes/`、`frontend/src/lib/components/`、`frontend/src/lib/stores/`、`frontend/src/lib/api/`、`frontend/src/app.css`  
> **验证命令**：`npm run check`、`npm run build`  
> **结论**：当前前端已经具备商业产品雏形，核心配音、多角色、字幕、音库、个人中心等页面完整度较高；下一阶段的主要优化不在“再加页面”，而在权限数据闭环、交互一致性、真实状态接入、性能拆包和页面工程治理。

---

## 一、总体判断

### 1.1 当前优势

| 方向 | 评价 |
|---|---|
| 产品结构 | 首页、配音、多角色、字幕、音库、文件、设置、个人中心、关于等菜单已完整成型 |
| 视觉完成度 | 暗色设计令牌、卡片化布局、波形播放器、三引擎表达已形成统一品牌感 |
| 配音主链路 | 文本输入、样音、引擎切换、参数、情感控制、生成、播放、下载路径清楚 |
| 多角色能力 | 角色、台词、行级情绪、批量生成、拼接导出，是当前最有商业差异化的页面 |
| 商业化 UI | 已有免费版 / 旗舰版 / 云端字符包 / 团队版的前端标识与升级弹窗雏形 |
| 安全意识 | LLM API Key 已走加密 store，Tauri 文件权限、CSP、字幕路径白名单已有基础 |

### 1.2 最大短板

当前前端最大问题是 **“UI 很像已商业化，但底层状态仍是演示态 / 多套状态并存”**。这会带来三个后果：

1. 用户看到的套餐、额度、云端余额、引擎可用性不一定来自同一个真实来源。
2. 部分付费能力已有锁标，但仍可通过快捷键、已有 demo 数据、行内控件绕过。
3. 页面规模已经较大，若继续堆功能，后续维护成本会迅速上升。

## 二、关键问题与优化点

### P0 · 权限状态是内存态，刷新即丢

**位置**：`frontend/src/lib/stores/membership.svelte.ts`

当前会员状态直接写在 `$state` 里：

```ts
let account = $state<MembershipAccount>({
  plan: 'free',
  dailyCharsUsed: 0,
  dailyQuota: 3000,
  cloudBalance: 0,
  devicesUsed: 1,
  devicesLimit: 1,
});
```

**风险**：

- 重启 / 刷新后套餐、今日额度、云端余额全部回到默认值。
- 与 `settings.svelte.ts` 里原有 `usage.quota = 500_000` 的月度额度体系并存，容易出现首页、个人中心、生成前校验显示不一致。
- 后续接支付时，如果继续沿用这套结构，会需要二次迁移。

**建议**：

1. 新建真实 `membership` 数据模型：`plan`、`entitlements`、`dailyUsage`、`cloudBalance`、`deviceBinding`、`lastSyncedAt`。
2. 本地 store 只做缓存，权威数据来自远程 `/user/membership`。
3. 免费额度计量从 `settings.usage` 与 `membership.account` 合并成单一来源。
4. 生成前统一走 `canGenerate(engine, chars)`，不要页面各自判断。

### P0 · 快捷键可绕过付费引擎锁

**位置**：`frontend/src/routes/+layout.svelte`

`Ctrl + Shift + E` 直接循环三引擎：

```ts
const idx = ENGINE_CYCLE.indexOf(dubbing.engineMode);
const next = ENGINE_CYCLE[(idx + 1) % ENGINE_CYCLE.length];
dubbing.setEngine(next);
toast.success(`已切换到${ENGINE_LABELS[next]}引擎`);
```

**风险**：

- 免费版用户可通过快捷键切到情感引擎 / 云端引擎。
- 虽然生成时会再次拦截，但 UI 状态已变成付费引擎，右侧情感控制也可能出现，造成权限口径混乱。

**建议**：

- 快捷键切换也必须走 `membership.canUseFeature()`。
- 若下一个引擎是锁定能力，弹出 `UpgradeModal`，不要改变 `dubbing.engineMode`。
- 把引擎切换封装为统一方法，例如 `requestEngineChange(mode)`，供标题栏快捷键、工具栏、设置页、多角色页共用。

---

### P0 · 多角色页行级编辑仍可绕过旗舰锁

**位置**：`frontend/src/routes/project/+page.svelte`

页面已给「行级编辑」加了旗舰标识，但行内控件仍直接改状态：

```svelte
<select class="lc-role-sel" ... onchange={(e) => { rolesStore.updateLine(...) }}>
<select class="lc-emo-sel" ... onchange={(e) => { rolesStore.updateLine(...) }}>
<select class="lc-str-sel" ... onchange={(e) => { rolesStore.updateLine(...) }}>
<input class="lc-text" ... onchange={(e) => { rolesStore.updateLine(...) }} />
```

**风险**：

- 免费版用户虽然看到锁标，但仍可直接改角色、情绪、强度、文本。
- 商业化边界不可信。

**建议**：

1. 行级控件统一加 `disabled={!membership.canUseFeature('line_editing')}`。
2. 点击锁定区域时弹升级弹窗。
3. 对 `rolesStore.updateLine()` 增加上层 guard，避免只靠 UI 禁用。
4. 删除角色、拖拽排序、重试失败项也应明确属于免费能力还是旗舰能力。

---

### P0 · 云端余额存在“双状态”

**位置**：`frontend/src/routes/setting/+page.svelte`

设置页显示云端余额时，同时使用了本地 `cloudBalance/cloudQuota` 和 `membership.account.cloudBalance`：

```svelte
<div class="balance-num">{membership.account.cloudBalance.toLocaleString('zh-CN')} 字</div>
<div class="balance-bar"><span style="width: {Math.min(100, (cloudBalance / cloudQuota) * 100)}%"></span></div>
```

**风险**：

- 数字和进度条可能不一致。
- 当前 `handleCloudLogin()` 实际只是跳转个人中心，`cloudLoggedIn` 很难进入真实已登录态。
- 生成前提示“余额充足”没有真实扣减链路支撑。

**建议**：

- 删除页面局部 `cloudBalance/cloudQuota/cloudLoggedIn`，统一从 `membership` 派生。
- 云端引擎可用性应由 `plan + cloudBalance + auth + backend health` 共同决定。
- 增加 `estimateCloudCost(text)` 和 `reserveCloudChars(jobId, chars)` 接口占位。

---

### P1 · 构建包体偏大，图标与页面未拆包

**验证结果**：`npm run build` 通过，但 Vite 警告：

- `Icon.js` 约 `729 KB`
- `XcMbUd_a.js` 约 `712 KB`
- `UUO4pa_R.js` 约 `311 KB`

**主要原因**：

- `Icon.svelte` 一次性 `addCollection(antDesignIcons)` 注册整个 Ant Design 图标集。
- 页面组件较大，`project`、`setting`、`resource`、`copywriting` 都是单文件巨页。

**风险**：

- Tauri 首屏 JS 解析和冷启动变慢。
- 后续继续加页面会恶化。

**建议**：

1. 图标改为构建期生成白名单 registry，只注册实际使用图标。
2. `project/+page.svelte` 拆成 `RolePanel`、`ScriptLineList`、`RoleParamPanel`、`ImportScriptModal`。
3. `setting/+page.svelte` 按 section 拆组件，并懒加载低频弹窗。
4. Vite 配置 `manualChunks`，把 iconify、tauri api、业务页拆开。

### P1 · 多角色页模板存在重复过滤，数据量大后会卡

**位置**：`frontend/src/routes/project/+page.svelte`

模板里多次对全量 `rolesStore.lines` 做 `filter`：

```svelte
{@const lineCount = rolesStore.lines.filter(l => l.roleId === role.id).length}
{@const doneCount = rolesStore.lines.filter(l => l.roleId === role.id && l.status === '已生成').length}
```

还有多处 `rolesStore.lines.filter(...)` 用在按钮 disabled、统计、失败项计算。

**风险**：

- 当前 15 行分页问题不大，但当用户导入几百 / 几千句台词时，角色列表、分页、状态变更会反复扫描全量数组。
- 批量生成过程中每句状态更新都会触发重复计算。

**建议**：

- 在 `rolesStore` 内新增派生索引：`lineStatsByRole`、`failedCount`、`generatedCount`。
- 模板只读派生结果，不在模板里做全量过滤。
- 批量生成期间减少逐句 `saveToStore()`，按批次或结束后落盘。

### P1 · “死按钮 / 演示数据”仍较多

**位置示例**：

- `about/+page.svelte`：授权管理、更新日志、打开数据目录、诊断日志仍是 toast。
- `cover/+page.svelte`：大量文件管理按钮是静态演示。
- `video/+page.svelte`：一键修正等按钮 disabled。
- `setting/+page.svelte`：多个 link disabled “即将推出”。

**风险**：

- 产品看起来已经可用，但点击后是“开发中”，会降低信任。
- 与付费升级入口混在一起时，用户会怀疑付费能力也不可用。

**建议**：

- 所有未实现能力统一进入 `FeaturePreview` 组件，显示「规划中 / 已解锁但未上线 / 需要升级」三种状态。
- 首页和工作台只放已可用功能；未上线功能放到「路线图」或「实验室」。
- 对已标价的能力，必须明确“已可用”或“即将开放”，不能只弹 toast。

### P1 · 全局隐藏滚动条影响可发现性

**位置**：`frontend/src/app.css`

全局隐藏所有滚动条：

```css
* {
  scrollbar-width: none;
  -ms-overflow-style: none;
}

*::-webkit-scrollbar {
  width: 0;
  height: 0;
  display: none;
}
```

多页面实际又使用内部滚动，如 `project`、`resource`、`about`、`video`。

**风险**：

- 用户不知道还有内容可滚动。
- 小屏幕下更明显。
- 与“单屏适配”页面规范冲突。

**建议**：

- 只隐藏主窗口滚动条，业务列表区域使用细滚动条或渐隐滚动条。
- 对长列表增加顶部 / 底部阴影提示可滚动。
- 保留 `data-scroll-area` 选择器，避免全局一刀切。

---

### P1 · 波形组件每次绘制都重设 canvas 尺寸

**位置**：`frontend/src/lib/components/ui/WaveformView.svelte`

`draw()` 中每次都会：

```ts
canvasEl.width = w * dpr;
canvasEl.height = h * dpr;
canvasEl.style.width = `${w}px`;
canvasEl.style.height = `${h}px`;
```

**风险**：

- 播放时 `currentTime` 高频变化，频繁重设 canvas 会清空上下文并触发布局 / 绘制开销。
- 多个播放器同时存在时，低配机器会明显。

**建议**：

- 尺寸变化和绘制分离：仅在 `containerWidth/height/dpr` 变化时重设 canvas。
- 播放进度变化时只重绘 bars / pointer。
- `computeWaveform()` 增加取消令牌，避免切换音频时旧 promise 回写。
- 为 slider role 补键盘左右键 seek。

---

### P2 · UI 组件规范执行不彻底

当前仍有大量原生 `button/select/input/textarea` 自写样式，尤其在：

- `project/+page.svelte`
- `resource/+page.svelte`
- `cover/+page.svelte`
- `copywriting/+page.svelte`
- `setting/+page.svelte`

**风险**：

- hover、focus、disabled、loading、aria 不一致。
- 后续主题切换、权限锁态、埋点会重复改很多地方。

**建议**：

- 保留极特殊业务控件，其余逐步迁移到 `Button/Input/Select/Empty/Modal`。
- 补一个通用 `Select` 和 `Segmented` 组件，替代各页面手写 select 和分段按钮。
- 新增 `LockedAction` 组件，统一锁态按钮 + 徽标 + 升级弹窗。

---

### P2 · 文件导入方式分散

多处使用 `document.createElement('input')`：

- `dubbing/+page.svelte`
- `project/+page.svelte`
- `TabEmotionControl.svelte`
- `resource/+page.svelte`

**风险**：

- 文件类型、大小限制、错误提示、Tauri 写入路径不统一。
- 后续做权限、审计、最近导入目录、拖拽导入时要改多处。

**建议**：

- 抽 `filePicker.ts`：`pickTextFile`、`pickAudioFile`、`pickMediaFile`。
- 统一限制：大小、扩展名、错误文案、最后打开目录。
- 桌面端优先用 Tauri dialog，浏览器 fallback 再用 input。

## 三、页面级优化建议

### 首页

- 快捷入口已展示锁态，但点击仍直接进入付费页。建议免费用户首次点击付费入口先弹 `UpgradeModal`，弹窗里保留“先看看示例”。
- 最近项目是示例数据，建议加更明显的「示例」空态逻辑；一旦有真实项目，隐藏示例。
- 引擎状态只检查本地健康，不应把云端引擎和本地健康混在同一个状态来源。

### 配音页

- 主链路完整，建议优先补“生成前成本卡片”：免费剩余额度 / 本地不扣字符 / 云端预计扣字。
- 情感控制锁态已补，但参数设置仍可打开高级参数。建议参数弹窗内按免费 / 旗舰分层禁用。
- 下载时应根据套餐决定 MP3 水印 / WAV 无损，而不是只做保存文件。

### 多角色页

- 商业价值最高，但也是目前最需要治理的页面。
- 建议先修 P0 权限绕过，再拆组件。
- 批量生成应支持失败原因列表，不只是“失败”状态。
- `confirm()` 删除应改 Modal，避免阻塞式原生弹窗破坏桌面体验。

### 字幕页

- ASR 基础流程清楚，翻译 / 时间轴优化已接后端。
- 建议把翻译和优化包装成“字幕增强”权益卡，用户可理解为什么要升级。
- 字幕行编辑目前是示例编辑，没有真实保存闭环，应避免误导。

### 音库页

- 预置音色、导入、人声分离形成了独立价值。
- 建议加入免费版「5 个自定义音色」限制，与付费方案对齐。
- `PRO` 已改「旗舰」，后续应区分“旗舰音色”和“用户导入音色”。

### 设置页

- 设置页功能多但信息密度偏高。
- 建议云端账号、LLM 智能拆分、引擎设置拆成独立子组件。
- “云端引擎”和 “LLM 智能拆分配置”不应都放在“云端账号”下，用户会混淆：一个是配音算力，一个是文本辅助。

### 个人中心

- 新版结构方向正确，已经能承载商业化。
- 需要下一步接真实会员接口，否则所有金额 / 额度 / 设备数都是演示。
- 套餐切换目前没有真实动作，建议按钮文案从“查看权益”改为“升级 / 购买 / 联系团队版”，并进入支付草稿弹窗。

## 四、建议实施顺序

### Sprint A · 权限闭环修正（最高优先）

1. 抽统一 `entitlementGuard.ts`：`canUseFeature`、`requireFeature`、`requestEngineChange`、`canGenerate`。
2. 修 `Ctrl+Shift+E` 绕过付费引擎。
3. 修多角色页行级编辑绕过。
4. 合并 `membership` 与 `settings.usage` 的额度来源。
5. 云端余额统一由 `membership` 派生。

### Sprint B · 商业化体验完善

1. 生成前成本提示：免费剩余 / 本地不限 / 云端预计扣字。
2. 下载导出接套餐差异：水印、WAV、MP3。
3. 音库自定义音色数量限制。
4. 字幕增强权益卡。
5. 个人中心支付弹窗原型。

### Sprint C · 页面工程治理

1. 拆 `project/+page.svelte`。
2. 拆 `setting/+page.svelte`。
3. 抽 `Select`、`Segmented`、`LockedAction`、`FilePicker`。
4. 消灭主要死按钮。

### Sprint D · 性能与包体

1. 图标白名单化，去掉全量 icon collection。
2. 页面动态拆包。
3. 优化 `WaveformView` canvas 绘制。
4. 给长列表补派生索引与虚拟列表预留。

## 六、结论

丸子前端已经过了“能不能做出来”的阶段，现在要进入“能不能稳定卖、稳定维护”的阶段。

最该先做的不是继续加新页面，而是把 **权限、额度、云端字符、套餐状态** 做成单一可信数据源；再把多角色页和设置页拆开，降低后续迭代成本。这样后续接支付、会员、字符扣费时，才不会推翻当前 UI。
