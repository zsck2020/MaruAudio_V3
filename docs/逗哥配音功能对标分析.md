# 逗哥配音功能对标分析

> 基于 https://www.douge.com/index 截图分析  
> 对标 IndexTTS v1.5（轻量引擎）和 v2.0（情感引擎）能力  
> 最后更新: 2026-02-16

---

## 一、功能清单与引擎能力评估

### 📝 文本处理类功能

| # | 功能 | 说明 | v1.5 | v2.0 | 实现方式 |
|---|------|------|:----:|:----:|---------|
| 1 | **导入文案** | 导入文本文件/剪贴板 | ✅ | ✅ | 前端功能，与引擎无关 |
| 2 | **文义段（语义分段）** | 按语义自动切分段落 | ✅ | ✅ | TextTokenizer.split_sentences() / split_segments()，两版本都有自动分句分段能力 |
| 3 | **全文读/文义段** | 全文直接合成 vs 分段合成 | ✅ | ✅ | v1.5 infer() 全文逐句 + infer_fast() 批处理；v2.0 infer_generator() 逐段生成 |
| 4 | **情感/情绪调节** | 调整语音的情感表达 | ❌ | ✅ | v2.0 独有：emo_vector 8维情感向量、emo_audio_prompt 情感参考音频、QwenEmotion 文本情感分析 |
| 5 | **感叹词** | 在文本中插入"嗯"、"啊"等语气词 | ⚠️ | ⚠️ | 引擎层面不直接支持，需在文本预处理阶段插入对应文字，引擎会自然合成。v1.5 曾有"嗯"合成 bug 已修复 |
| 6 | **宣传配音** | 特定语气风格（激昂、大气） | ❌ | ✅ | v2.0 可通过情感向量或风格参考音频实现。v1.5 只能通过选择合适的参考音频 |
| 7 | **真实感** | 让语音更接近真人说话 | ✅ | ✅ | 两版本都是零样本语音克隆，天然具备真实感。可通过 temperature、top_p 等参数调节自然度 |
| 8 | **数字读法** | 数字转中文朗读（123→一百二十三）| ✅ | ✅ | TextNormalizer 内置数字转换（WeTextProcessing），两版本共有 |
| 9 | **儿化音** | 支持儿化音（花儿、哪儿）| ✅ | ✅ | BPE 分词 + 拼音混合建模，训练数据中包含儿化音样本 |

### 🎤 语音控制类功能

| # | 功能 | 说明 | v1.5 | v2.0 | 实现方式 |
|---|------|------|:----:|:----:|---------|
| 10 | **自动配音** | 一键生成语音 | ✅ | ✅ | 核心功能 infer() / infer_fast() |
| 11 | **自动分割** | 长文本自动分割为多段 | ✅ | ✅ | split_sentences(max_tokens=100~120) / split_segments() |
| 12 | **局部变速** | 对某一段文字单独调速 | ❌ | ❌ | **两版本都不直接支持**。需后处理实现：分段生成后对特定段用 FFmpeg/torchaudio 变速，或通过插入标点控制停顿 |
| 13 | **弹幕区** | 生成弹幕效果的配音 | N/A | N/A | 前端展示功能，与引擎无关 |
| 14 | **节拍/节奏** | 调整语音的节奏感 | ⚠️ | ⚠️ | 引擎不直接暴露节拍控制。可通过：标点控制停顿、temperature 影响韵律、v2.0 的 interval_silence 控制段间停顿 |
| 15 | **拼音标注** | 为多音字标注正确读音 | ✅ | ✅ | **核心特色**！中文字符-拼音混合建模，如 `晕 XUAN4` 指定发音。两版本共有 |
| 16 | **方言/粤语** | 支持方言合成 | ❌ | ❌ | **两版本都不支持方言**。IndexTTS 训练数据主要为普通话+英语，不含粤语/四川话等方言。需要额外方言模型 |
| 17 | **声调标记** | 标注声调（一声~四声）| ✅ | ✅ | 拼音标注自带声调，如 MA1=妈, MA2=麻, MA3=马, MA4=骂 |
| 18 | **人声入驻** | 上传自己的声音作为音色 | ✅ | ✅ | 零样本语音克隆的核心能力：上传参考音频即可克隆音色，两版本都支持 |
| 19 | **多人对话** | 多角色对话配音 | ✅ | ✅ | 前端分配角色 + 每段使用不同参考音频生成。引擎本身支持切换参考音频（有缓存优化） |

### 🔊 声音库与参数控制

| # | 功能 | 说明 | v1.5 | v2.0 | 实现方式 |
|---|------|------|:----:|:----:|---------|
| 20 | **声音库分类** | 按风格分类声音（甜白、解说、情感...）| ✅ | ✅ | 前端+后端功能：管理预录的参考音频库，按标签分类。引擎只需接收参考音频路径 |
| 21 | **语速调节（0.5~1.7x）** | 控制语速 | ⚠️ | ⚠️ | 引擎不直接暴露语速参数。实现方式：(1) 后处理 FFmpeg 变速不变调 (2) v2.0 有 use_speed 参数（代码中存在但未完全暴露） (3) 通过 temperature 间接影响 |
| 22 | **情感/风格标签** | 为声音标记情感风格 | N/A | ✅ | 前端标签系统。v2.0 可通过 QwenEmotion 自动检测文本情感 |
| 23 | **收藏声音** | 收藏常用声音 | ✅ | ✅ | 纯前端功能 |
| 24 | **生成字幕** | 从音频生成字幕 | ❌ | ❌ | **引擎不提供 ASR 功能**。需接入独立的 ASR 服务（如必剪 ASR、Whisper） |
| 25 | **下载视频** | 导出带字幕的视频 | N/A | N/A | 前端+FFmpeg 后处理，与引擎无关 |

## 二、关键差距分析

### 🔴 完全不支持，需额外方案

| 功能 | 差距说明 | 建议方案 |
|------|---------|---------|
| **方言/粤语** | IndexTTS 无方言训练数据 | 接入阿里云百炼等云端 API 支持方言（已在开发方案中规划为"云端引擎"） |
| **精确语速控制** | 引擎无直接语速参数 | 方案 1: FFmpeg tempo 后处理变速不变调；方案 2: v2.0 的 duration control（论文提到但当前版本未启用） |
| **局部变速** | 分段级变速 | 分段生成 → 对指定段 FFmpeg 变速 → 合并 |
| **ASR 字幕生成** | 引擎无 ASR 功能 | 接入 Whisper / 必剪 ASR / 阿里云 ASR 独立服务 |

### 🟡 部分支持，需增强

| 功能 | 差距说明 | 建议方案 |
|------|---------|---------|
| **情感调节** | v1.5 不支持，v2.0 支持 | v1.5 通过选择不同参考音频间接实现；v2.0 直接使用 emo_vector |
| **感叹词** | 引擎不主动生成语气词 | 文本预处理阶段自动/手动插入语气词文字 |
| **节拍/节奏** | 无直接参数 | 标点控制停顿 + temperature 调节 + interval_silence |
| **宣传配音** | 需特定语气 | v2.0 情感向量 + 激昂参考音频；v1.5 仅靠参考音频 |

### 🟢 完全支持

| 功能 | 说明 |
|------|------|
| **导入文案** | 前端功能 |
| **语义分段** | TextTokenizer 内置 |
| **数字读法** | TextNormalizer 内置 |
| **拼音标注** | 核心特色，字符-拼音混合建模 |
| **声调标记** | 拼音体系自带 |
| **儿化音** | 训练数据包含 |
| **人声入驻** | 零样本克隆核心能力 |
| **多人对话** | 切换参考音频即可 |
| **自动配音** | 核心 infer 功能 |
| **自动分割** | 内置分句分段 |
| **声音库** | 前端 + 参考音频管理 |
| **真实感** | 零样本克隆天然特性 |

## 四、丸子配音实现建议

### 第一优先级（核心功能）
1. **文本配音生成** — v1.5/v2.0 均可
2. **语义分段 + 自动分割** — 引擎内置
3. **拼音标注 + 声调** — 引擎内置
4. **数字读法** — 引擎内置
5. **声音库管理** — 前端实现，参考音频分类存储
6. **人声入驻（上传音色）** — 零样本克隆

### 第二优先级（差异化功能）
7. **情感/情绪调节** — 使用 v2.0 引擎
8. **多人对话** — 前端角色分配 + 切换参考音频
9. **语速调节** — FFmpeg 后处理 tempo
10. **宣传配音风格** — v2.0 情感向量 + 风格参考音频

### 第三优先级（扩展功能）
11. **字幕生成** — 接入独立 ASR 服务
12. **方言支持** — 接入云端引擎（阿里云百炼 Qwen3-TTS）
13. **局部变速** — 分段生成 + 后处理
14. **感叹词/语气词** — 文本预处理插入

### 引擎选择策略

| 场景 | 推荐引擎 | 理由 |
|------|---------|------|
| 短文本快速生成 | v1.5 | 速度快，2步管线 |
| 长文本批量生成 | v1.5 (infer_fast) | 桶化批处理，2~10x 加速 |
| 情感丰富的配音 | v2.0 | 8维情感控制 |
| 宣传片/广告配音 | v2.0 | 风格解耦，情感表达强 |
| 方言配音 | 云端引擎 | 本地引擎不支持 |
| 低端设备 | v1.5 / 云端 | v2.0 VRAM 需求高 |
