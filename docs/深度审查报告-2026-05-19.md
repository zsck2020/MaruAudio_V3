### #1 · envs/v15 + envs/v20 requirements.txt 完全没有内容

- `envs/v15/requirements.txt`（2 行注释 · 0 个 pip 依赖）

# IndexTTS v1.5 轻量引擎依赖
# 安装: cd envs/v15 && python -m venv .venv && .venv\Scripts\activate && pip install -r requirements.txt

**后果**：
- 任何接手的开发或 AI 按 README 走 `pip install -r requirements.txt` 装不出任何包

- **环境不可重建** = 项目无法被其他人/CI/打包流程消费
- commit `57e185f` 改了这两个文件（diff 显示 +2 lines），但只是改了注释、没补依赖

**修法**：
```txt
# envs/v15/requirements.txt（v1.5 引擎需要）
indextts==1.5.0  # 或 git+ssh 引用
torch>=2.1.0
torchaudio>=2.1.0
... 等等
```
需要参考 `IndexTTS/v15/requirements.txt`（如有）或 `IndexTTS/v15/pyproject.toml` 来确定真实依赖。

### #2 · models.py 的 engine 字段 pattern 不含 "cloud"

**位置**：`backend/maruaudio_tts/models.py:14`

```python
engine: str = Field(default="lightweight", pattern=r"^(lightweight|emotion)$")
```

**后果**：
- server.py 第 80-99 行的 `synthesize` 函数有 `if req.engine == "cloud"` 分支
- server.py 第 101-115 行的 `synthesize_sse` 函数同样
- 但 **Pydantic 校验在反序列化时就会拒绝 engine="cloud" 的请求**，返回 422 Unprocessable Entity
- → **云端推理永远走不到 cloud_engine 分支**
- 而 cloud_engine.py 本身是好的 · engine_manager 也支持 `"cloud"` 字面量

**修法**：
```python
engine: str = Field(default="lightweight", pattern=r"^(lightweight|emotion|cloud)$")
```

## 八、整体结论

**整体评价**：项目**主干工程素质很高**（特别是 auth/crypto/storage 层、UI 原子库、安全策略），但**业务页面有大量"半完成"占位**（云端任务、PresetLibrary、handle* 占位、TitleBar mock 数据），且**有 4 个严重问题阻塞了"项目可被他人/CI 消费"**（envs 全空、models 拒 cloud、lib/subtitle 孤儿、Python 0 测试）。

**最大风险**：
- **可复现性**：envs requirements 空 → 任何人都装不出环境
- **云端推理实际跑不通**：models.py engine pattern 缺 cloud（这是个"以为修好其实没修"的硬 bug，需要立刻确认）

**最大优势**：
- 主干工程**敢称专业级**（crypto/storage/api/user 模块和 UI 原子层）
- 安全策略组合（CSP/CORS/token 加密/keyring/禁词）执行到位
- 配音业务核心链路（参考音频上传 → 人声分离 → 推理 → 播放器）已贯通

**建议**：先 1 小时把【严重】4 项搞定（解锁可复现性 + 云端推理 + 删孤儿代码 + Python 测试起步），后续按猫总优先级慢慢推进高/中/低。
